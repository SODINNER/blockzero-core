# Contributing

Thanks for helping build Block Zero.

## Ground Rules

- Keep changes small and reviewable.
- Prefer issues before large PRs.
- Keep communication technical and transparent.
- Avoid investment language in docs or comments.

## Development Workflow

1. Fork and create a feature branch.
2. Add tests/docs for user-facing or consensus-impacting changes.
3. Open a PR with clear rationale and risk notes.
4. Wait for review before merge.

## Commit Style

Use concise, intent-focused messages:

- `add ...`
- `update ...`
- `fix ...`
- `docs ...`
